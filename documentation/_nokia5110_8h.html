Detailed Description

Provide simple I/O functions for the Nokia 5110.

Use eUSCI_A3 to send an 8-bit code to the Nokia5110 48x84 pixel LCD to display text, images, or other information.

void Nokia5110_Init ( void  )

Initialize LCD driver.

Initialize Nokia 5110 48x84 LCD by sending the proper commands to the PCD8544 driver. One feature of the MSP432 is that its SSIs can get their baud clock from either the auxiliary clock (ACLK = REFOCLK/1 = 32,768 Hz see Clock.c) or from the low-speed subsystem master clock (SMCLK <= 12 MHz see Clock.c). The SSI can further divide this clock signal by using the 16-bit Bit Rate Control prescaler Register, UCAxBRW.

Parameters
none
Returns
none
Note
Assumes low-speed subsystem master clock is 12 MHz

◆ Nokia5110_OutChar()

void Nokia5110_OutChar ( char  data)

Print a character to the LCD.

Print a character to the Nokia 5110 48x84 LCD. The character will be printed at the current cursor position, the cursor will automatically be updated, and it will wrap to the next row or back to the top if necessary. One blank column of pixels will be printed on either side of the character for readability. Since characters are 8 pixels tall and 5 pixels wide, 12 characters fit per row, and there are six rows.

Parameters
datacharacter to print
Returns
none
Note
LCD is in default horizontal addressing mode (V = 0)

Print a 32-bit unsigned number to the LCD.

Print a number in signed decimal format to the Nokia LCD.

Parameters
n32-bit unsigned number
min_lengthminimum length to print. If min_length is less than or equal to the length of the string, min_length will be ignored. If it is greater than the length of the string, spaces will be added in front of the string.
Returns
none
Note
LCD is in default horizontal addressing mode (V = 0)
See also
Nokia5110_Init(), Nokia5110_Clear(), Nokia5110_SetCursor()
Note
If the integer is -12345 and min_length = 7, it will print " -12345". Notice that a space has been added to the front. If the integer is -12345 and min_length <= 6, it will print "-12345". If the integer is 12345 and min_length <=6, it will print " 12345". Notice there is a space at the beginning, which makes consistent with Nokia5110_OutS32Dec, Nokia5110_OutS16Dec, and Nokia5110_OutS8Dec.

Note: You can simply set min_length = 0 if you like to print the actual length of the string.

Put 48x84 unformatted 4-bit color bitmap image in internal screen buffer.

Bitmaps defined above were created for the LM3S1968 or LM3S8962's 4-bit grayscale OLED display. They also still contain their header data and may contain padding to preserve 4-byte alignment. This function takes a bitmap in the previously described format and puts its image data in the proper location in the buffer so the image will appear on the screen after the next call to Nokia5110_DisplayBuffer(); The interface and operation of this process is modeled after RIT128x96x4_BMP(x, y, image);

Parameters
xposhorizontal position of bottom left corner of image, columns from the left edge
must be less than 84
0 is on the left; 82 is near the right
yposvertical position of bottom left corner of image, rows from the top edge
must be less than 48
2 is near the top; 47 is at the bottom
ptrpointer to a 16 color BMP image
thresholdgrayscale colors above this number make corresponding pixel 'on'
0 to 14
0 is fine for ships, explosions, projectiles, and bunkers
Returns
none
Note
Call Nokia5110_DisplayBuffer() to see this bitmap.
See also
Nokia5110_ClearBuffer(), Nokia5110_DisplayBuffer(), Nokia5110_SetPxl()
